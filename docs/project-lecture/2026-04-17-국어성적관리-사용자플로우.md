# 국어 성적 관리 & 상담 준비 도구 — 사용자 플로우

## 전체 흐름 요약

```
[1. 학생 등록] → [2. 시험 생성] → [3. 점수 입력] → [4. 분석 확인] → [5. 상담문 생성] → [6. 상담 후 메모]
     ↑                                                                                          |
     └─────────────────────── 다음 시험 주기 ──────────────────────────────────────────────────────┘
```

## 플로우 상세

### 플로우 1. 초기 세팅 (최초 1회)

```
강사 로그인
  → 반 생성 (예: "고1-A", "고2-B" ... 7개)
  → 반별 학생 등록 (이름, 학교)
  → 완료
```

**핵심 UX:** 70명 등록이 번거롭지 않아야 함.
- 엑셀/CSV 일괄 업로드 지원 권장
- 또는 "이름, 학교명" 한 줄씩 빠른 입력

**온보딩:** 최초 접속 시 빈 화면이면 막막할 수 있음.
- 샘플 반·학생 데이터를 미리 넣어두고 "체험해보기" 제공 권장
- 또는 "1. 반 만들기 → 2. 학생 추가 → 3. 첫 시험 입력" 3단계 가이드 투어

### 플로우 2. 시험 후 점수 입력 (시험마다 반복)

```
대시보드
  → "새 시험 추가" 클릭
  → 시험 정보 입력
     - 시험 유형: [학교 중간] [학교 기말] [모의고사]
     - 시험 날짜
     - 학년/반 선택
  → 학생별 점수 입력
     - 총점
     - 영역별 세부 점수 (문법, 독해, 문학 등)
  → 저장
```

**핵심 UX:** 70명 점수 입력이 빨라야 함.
- 표 형태로 한 화면에서 탭 키로 연속 입력
- 영역별 점수는 선택 입력 (총점만 넣어도 기본 분석 가능)

### 플로우 3. 성적 분석 확인

```
학생 목록에서 학생 클릭
  → 학생 상세 페이지
     ├── 성적 추이 차트 (시험 회차별 점수 변화 그래프)
     ├── 이번 시험 분석
     │   - 이전 시험 대비 변화 (+/- 점)
     │   - 반 평균 대비 위치
     │   - 영역별 강약점 (입력된 경우)
     ├── 시험 유형별 비교 (학교시험 vs 모의고사 추이)
     └── 메모 이력
```

**핵심 UX:** 상담 직전에 한눈에 파악할 수 있어야 함.
- 차트는 한 화면에 핵심만
- 숫자보다 **변화 방향**(상승/하락/유지)이 먼저 눈에 들어오게

### 플로우 4. AI 상담문 생성

```
학생 상세 페이지에서
  → "상담문 생성" 버튼 클릭
  → AI가 자동 생성 (2~5초)
     - 입력: 이번 점수, 이전 점수들, 반 평균, 영역별 점수, 기존 메모
     - 출력: 학부모 대상 설명문 초안
       ├── 전체 성적 변화 요약
       ├── 강점 (잘하는 영역)
       ├── 보완점 (약한 영역)
       └── 제안 (다음 시험까지 할 일)
  → 강사가 초안 확인
     - 수정 필요 시: 직접 편집 후 저장
     - OK면: 그대로 저장
  → 저장된 상담문은 학생 이력에 누적
```

**핵심 UX:** 생성된 문장이 자연스럽고, 교사 어투에 맞아야 함.
- "~입니다" 존댓말 학부모 소통 톤
- 막연한 격려("잘하고 있습니다")보다 **구체적 근거**("문학 영역이 지난 시험 대비 12점 상승")

### 플로우 5. 상담 후 메모

```
상담 완료 후
  → 학생 상세 페이지에서 "메모 추가"
  → 특이사항 기록
     - 학부모 요청사항
     - 학생 상태/태도 변화
     - 다음 상담 때 확인할 점
  → 저장 (다음 상담문 생성 시 AI가 참조)
```

**핵심 UX:** 간단히 메모할 수 있어야 함.
- 긴 양식 X, 자유 텍스트 입력
- 이전 메모 이력이 시간순으로 보임

## 반복 주기

| 시점 | 하는 일 | 예상 소요 |
|------|---------|-----------|
| 학기 초 | 플로우 1 — 학생 등록 (또는 이전 학기에서 이월) | 30분~1시간 |
| 시험 후 (학기당 4회+) | 플로우 2 — 점수 입력 | 반당 10~15분 |
| 상담 준비 | 플로우 3+4 — 분석 확인 + 상담문 생성 | **학생당 2~3분** |
| 상담 후 | 플로우 5 — 메모 기록 | 학생당 1~2분 |

## 핵심 설계 원칙

1. **점수 입력이 가장 빨라야 한다** — 가장 자주 하는 행위. 탭 키 연속 입력, 일괄 업로드.
2. **분석은 자동, 확인은 한눈에** — 강사가 계산하지 않는다. 열면 이미 분석이 되어 있다.
3. **상담문은 초안이지 완성본이 아니다** — AI 결과를 강사가 검토·수정하는 구조. 100% 자동화를 목표로 하지 않는다.
4. **데이터는 누적된다** — 이전 시험, 이전 상담 메모가 다음 상담문에 자동 반영. 쓸수록 정확해진다.
