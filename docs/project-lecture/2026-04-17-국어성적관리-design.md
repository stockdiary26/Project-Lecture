# 국어 성적 관리 & 상담 준비 도구 — 기술 설계서

## 개요

국어학원 강사가 학생별 성적을 입력하면 자동 분석 + AI 상담문 초안을 생성하는 웹 앱.
학생당 상담 준비 시간을 20분 → 2~3분으로 줄이는 것이 목표.

- **사용자:** 국어학원 강사 1인
- **플랫폼:** 웹 앱 (Streamlit Cloud 배포, 브라우저 접속)
- **기획 문서:** `docs/product/2026-04-17-국어성적관리-기획서.md`

---

## 기술 스택

| 계층 | 기술 | 선택 이유 |
|------|------|-----------|
| 프론트+백 | **Streamlit** | Python만으로 완성. 초보자 학습 곡선 최소화 |
| DB | **Supabase (PostgreSQL)** | 무료 클라우드 DB. Streamlit Cloud에서 SQLite 영속 불가하므로 |
| AI | **Claude API** | 상담문 생성. 종량제 연 $4~8 수준 |
| 차트 | **Plotly (Streamlit 내장)** | 인터랙티브 차트. 추가 학습 불필요 |
| 배포 | **Streamlit Cloud** | 무료. GitHub 연동 자동 배포 |
| 테스트 | **pytest** | Python 표준 테스트 프레임워크 |

## 데이터베이스 설계

### ERD

```
classes (반)
├── id (PK)
├── name              -- "고1-A", "고2-B" 등
└── created_at

students (학생)
├── id (PK)
├── class_id (FK → classes)
├── name
├── school_name       -- 학교명
└── created_at

categories (영역 카테고리)
├── id (PK)
├── name              -- "문법", "독해", "문학" 등
├── sort_order        -- 표시 순서
└── created_at

exams (시험)
├── id (PK)
├── name
├── exam_type         -- '중간', '기말', '모의고사'
├── exam_date
└── created_at

scores (점수)
├── id (PK)
├── student_id (FK → students)
├── exam_id (FK → exams)
├── total_score       -- 총점 (필수, 0~100)
├── grade             -- 내신등급 (선택, 1~9)
└── created_at

category_scores (영역별 점수)
├── id (PK)
├── score_id (FK → scores)
├── category_id (FK → categories)
└── score             -- 영역별 점수 (선택)

consultations (상담문)
├── id (PK)
├── student_id (FK → students)
├── exam_id (FK → exams)
├── ai_draft          -- AI 생성 초안
├── final_text        -- 강사 수정본
└── created_at

memos (메모)
├── id (PK)
├── student_id (FK → students)
├── content           -- 자유 텍스트
└── created_at
```

### 입력 검증 규칙

| 필드 | 필수/선택 | 검증 |
|------|----------|------|
| total_score | **필수** | 0~100 범위. 유일한 필수 입력 |
| grade | 선택 | 입력 시 1~9 범위. 빈 칸 허용 |
| category_scores.score | 선택 | 입력 시 0 이상. 빈 칸 허용 |

**원칙:** 총점만 있으면 저장 가능. 나머지는 있으면 분석에 활용, 없으면 해당 분석을 건너뜀.

## 페이지 구성

### 사이드바 메뉴

```
├── 🏠 대시보드        -- 전체 요약 (홈)
├── 👩‍🎓 학생 관리      -- 반/학생/학교명 CRUD
├── 📝 성적 입력       -- 시험 생성 + 점수 입력
├── 📊 성적 분석       -- 학생별 분석 + 차트
├── 💬 상담문 생성     -- AI 상담문 생성/편집
├── 📋 메모            -- 학생별 메모 관리
└── ⚙️ 설정           -- 영역 카테고리 관리
```

### 페이지별 상세

#### 1. 대시보드

- 최근 시험 요약 카드
- "상담문 미작성 학생 N명" 등 할 일 안내

#### 2. 학생 관리

- 반 생성/수정/삭제
- 학생 추가/수정/삭제 (이름, 학교명)
- CSV 일괄 업로드
- **필터:** 반별 보기 / 학교별 보기 전환
- 학교별 보기 시 학교명 기준 그룹핑, 학생 수 표시

#### 3. 성적 입력

- 시험 생성: 유형(중간/기말/모의고사), 날짜, 반 선택
- 표 형태 입력 (`st.data_editor`): 탭 키로 연속 입력
- 열: 학생명 | 학교명 | 총점 | 등급 | 영역1 | 영역2 | ...
- 학교별 소팅 토글
- CSV 일괄 입력 지원

#### 4. 성적 분석

- 학생 선택 (반별/학교별 그룹 전환)
- 성적 추이 차트 (Plotly 라인 차트, 시험 회차별)
- 이번 시험 분석:
  - 이전 시험 대비 총점 변화 (+/- 점)
  - 반 평균 대비 위치
  - 영역별 강약점
  - 영역별 이전 시험 대비 변화
- 시험 유형별 비교 (학교시험 vs 모의고사 추이)
- 학교별 평균 비교

#### 5. 상담문 생성

- 학생 + 시험 선택
- "상담문 생성" 버튼 → Claude API 호출 (2~5초)
- AI 초안 표시 → 수정 가능 → 저장
- 재생성 버튼
- 저장된 상담문은 학생 이력에 누적

#### 6. 메모

- 학생 선택 → 메모 추가 (자유 텍스트)
- 시간순 메모 이력 표시
- 다음 상담문 생성 시 AI가 메모를 참조

#### 7. 설정

- 영역 카테고리 추가/수정/삭제
- 드래그 또는 버튼으로 순서 변경

## AI 상담문 생성 설계

### 데이터 수집 (자동)

상담문 생성 버튼 클릭 시 DB에서 자동 수집:
- 이번 시험 총점/등급/영역별 점수
- 이전 시험들 점수 이력
- 영역별 이전 대비 변화
- 반 평균 / 학교별 평균
- 이전 상담문 (있으면)
- 학생 메모 (있으면)

### 프롬프트 설계

| 요소 | 내용 |
|------|------|
| 톤 | 학부모 대상 존댓말, 전문적이되 따뜻한 어조 |
| 구조 | 1) 전체 성적 변화 요약 → 2) 강점 영역 → 3) 보완 영역 → 4) 구체적 제안 |
| 근거 | 모든 언급에 숫자 근거 포함 ("문학 영역 28→35점, 7점 상승") |
| 맥락 | 이전 상담문·메모 있으면 연속성 유지 |
| 길이 | 300~500자 |

### 비용

| 항목 | 추정 |
|------|------|
| 1회 호출당 | ~1,500 토큰 (~$0.01) |
| 시험 1회 (70명) | ~$0.5~1.0 |
| 연간 8회 | ~$4~8/년 |

## 에러 처리

| 상황 | 처리 |
|------|------|
| Claude API 호출 실패 | "상담문 생성에 실패했습니다" + 재시도 버튼. 나머지 기능은 정상 동작 |
| API 키 미설정 | 설정 페이지로 안내. 입력/분석/차트는 AI 없이 독립 작동 |
| 총점 범위 초과 | 0~100 범위 검증, 저장 차단 |
| 등급 범위 초과 | 입력 시 1~9 검증. 빈 칸은 정상 |
| DB 연결 실패 | "데이터 서버에 접속할 수 없습니다" + 자동 재시도 3회 |
| 영역별 점수 합 ≠ 총점 | 경고 표시하되 저장 허용 (배점 구조가 시험마다 다를 수 있음) |

**원칙:** AI 없이도 입력·분석·차트는 독립 작동. API 장애가 전체 앱을 막지 않음.

| 계층 | 대상 | 방법 |
|------|------|------|
| 데이터 | DB CRUD, 점수 계산, 평균/변화량 로직 | pytest 단위 테스트 |

## 인증

- Streamlit 앱 시작 시 비밀번호 입력 페이지
