# 성적 입력 — 학생별/시험별 두 모드 통합 — 설계

## 배경

기존 `pages/2_📝_성적입력.py`는 다음 구조:
1. 시험 선택/생성
2. 반 단위로 학생들의 **총점·등급**만 데이터 편집기로 입력
3. 하단에 별도 섹션으로 **영역별 점수**를 학생 1명씩 입력

제약:
- 영역별 점수가 총점 입력과 분리돼 있어 한 학생의 완전한 성적을 입력하려면 두 섹션을 오가야 함
- 시험별 관점만 제공 — 한 학생의 전 시험 추이를 손쉽게 업데이트할 방법이 없음

## 목표

한 페이지에서 **학생별 / 시험별** 두 가지 관점으로 입력할 수 있고, 각 모드 모두에서 **총점·등급·영역별 점수**를 하나의 표에서 입력·저장할 수 있게 한다.

## 페이지 구조

```
📝 성적 입력
├─ [공용] 시험 선택/생성
└─ 탭
    ├─ 👤 학생별 입력
    └─ 📝 시험별 입력
```

`st.tabs([...])`로 구현.

## 탭 1: 학생별 입력

**UI 흐름**
1. 반 선택 (선택적 필터) → 학생 선택
2. 해당 학생의 **모든 기존 시험 행** + 점수 있는 경우 현재 값 채움
3. 단일 `st.data_editor`로 편집 → '저장' 버튼

**컬럼**
| 고정(읽기 전용) | 편집 |
|---|---|
| 시험명 · 유형 · 날짜 | 총점 · 등급 · <카테고리 1..N> |

`<카테고리>`는 `db.get_categories()`로 동적 생성 — 설정 페이지에서 카테고리를 추가하면 자동 반영.

**행 원본**: `db.get_scores_by_student(student_id)`에 없는 시험은 빈 행으로 포함하기 위해, `db.get_exams()`와 left-join 방식으로 병합.

## 탭 2: 시험별 입력

**UI 흐름**
1. 반 선택 → 시험 선택
2. 반의 **모든 학생 행** + 점수 있는 경우 현재 값 채움
3. 단일 `st.data_editor`로 편집 → '저장' 버튼

**컬럼**
| 고정(읽기 전용) | 편집 |
|---|---|
| 이름 · 학교명 | 총점 · 등급 · <카테고리 1..N> |

## 저장 로직 (두 탭 공통)

```
각 편집된 행에 대해:
  if 총점 is None:
    continue   # 총점은 필수 — 행 전체 스킵
  if not (0 <= 총점 <= 100):
    append error, continue
  if 등급 is not None and not (1 <= 등급 <= 9):
    append error, continue

  score_row = db.upsert_score(student_id, exam_id, 총점, 등급)

  for 카테고리 in 카테고리_목록:
    값 = row[카테고리]
    if 값 is None or 값 == "":
      continue   # 빈 영역은 스킵 (기존 영역 점수는 유지)
    db.upsert_category_score(score_row["id"], 카테고리.id, 값)
```

**정책 확정**
- **총점**: 필수. 비어 있으면 해당 행 전체 스킵 (영역별 포함).
- **등급**: 선택. 비어 있어도 저장 계속.
- **영역별**: 각 셀 독립. 값 있으면 upsert, 없으면 해당 셀 스킵.
- **기존 영역별 점수 삭제는 이번 범위 밖.** 값을 비워 저장해도 기존 DB 레코드는 유지됨. (필요 시 추후 '영역 지우기' UI 추가)

- 탭 1 데이터 편집기 key: `"editor_by_student_{student_id}"`
- 탭 2 데이터 편집기 key: `"editor_by_exam_{exam_id}_{class_id}"`
- 각 selectbox: 기존 명명 규칙 따름

## 검증 · 테스트

- 로직 대부분이 Streamlit UI·DB I/O이므로 기존 단위 테스트는 영향 없음.
- 회귀 방지 체크리스트 (수동):
  1. 학생별 입력 → 총점만 입력하고 저장 → scores 테이블에 upsert 확인
  2. 학생별 입력 → 총점+영역별 입력하고 저장 → scores + category_scores 저장 확인
  3. 시험별 입력 → 여러 학생 일괄 저장
  4. 카테고리 추가/삭제 후 페이지 새로고침 → 컬럼이 자동 반영
  5. 총점 빈 행은 오류 없이 스킵
